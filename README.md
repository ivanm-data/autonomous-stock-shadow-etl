# 📦 Autonomous Stock Shadow ETL

**Autonomous Stock Shadow ETL (ASSE)** — это автономная система теневого мониторинга, аналитики и управления складскими остатками. Проект вырос из простого ETL-пайплайна в полноценную гибридную ERP-систему, которая использует алгоритмы ИИ и машинного зрения для автоматизации складского учета, приемки товаров и выявления аномалий.

## ✨ Ключевые возможности

### 🤖 ИИ-Приемка и Оцифровка (AI Receiving)
* **Распознавание накладных:** Загрузка фотографий бумажных накладных. Нейросеть (Gemini/GPT-4o) автоматически оцифровывает табличные данные (названия, артикулы, количество).
* **Fuzzy Matching (Нечеткий поиск):** Интеллектуальное сопоставление оцифрованных строк с реальными аномалиями на складе. Если ИИ ошибся в одной букве при чтении бумаги (например, `0` вместо `O`), система всё равно найдет совпадение и предложит склеить позиции одной кнопкой.

### ⚠️ Интеллектуальный поиск аномалий (Anomaly Detection)
* **Паттерн «Inbox»:** Ежедневные скачки остатков (пропажи, излишки, пересорт) не теряются при смене дня. Они сохраняются в персистентный буфер и ждут ручной модерации или автоматической легализации.
* **Трекинг истории товаров:** Система автоматически распознает 5 состояний: `НОВИНКА`, `ДОВОЗ`, `ВОЗВРАТ`, `СМЕНИЛСЯ АРТИКУЛ` и `СМЕНИЛОСЬ ИМЯ`.
* **Умная склейка (Item Aliases):** Возможность в один клик "склеить" старую и новую карточку товара, если менеджеры на сайте изменили название или артикул. Вся историческая аналитика при этом сохраняется.

### 📊 Складская аналитика
* **Оборачиваемость (Velocity):** Построение исторических графиков движения по каждому конкретному товару (даже с учетом его переименований в прошлом).
* **Анализ неликвида (Dead Stock):** Автоматический расчет «замороженных» товаров на основе медианного времени без движения внутри каждой категории.
* **Data Engineering (Write-Time Processing):** Сверхбыстрый поиск по базе в сотни тысяч строк за счет предварительной нормализации текста в ETL-процессе и составных индексов в SQLite.

## 🏗 Архитектура

Проект разделен на независимые слои:
1. **ETL & Crawler (`parser.py`):** Автономные скрипты сбора данных, работающие по расписанию.
2. **Database & Cache (`db.py` / SQLite):** Легкая, но мощная локальная БД. Хранит историю (stocks), логи аномалий (anomaly_log), буферы накладных (expected_deliveries) и связи (item_aliases).
3. **AI Services (`ai_services.py`):** Модуль общения с LLM-моделями для анализа данных и OCR.
4. **Frontend / UI (`src/main.py`, `src/nice_views/`):** Интерактивный дашборд. (В процессе миграции со `Streamlit` на событийно-ориентированный `NiceGUI` для максимального быстродействия).

## 🚀 Установка и запуск

### Требования
* Python 3.10+
* Установленные зависимости из `requirements.txt`

### Установка
1. Клонируйте репозиторий:
```
git clone [https://github.com/ivanm-data/autonomous-stock-shadow-etl.git](https://github.com/ivanm-data/autonomous-stock-shadow-etl.git)
cd autonomous-stock-shadow-etl
```

2. Создайте виртуальное окружение и установите зависимости:
```
python -m venv venv
venv\Scripts\activate  # Для Windows
pip install -r requirements.txt
```

3. Скопируйте файл конфигурации:
```
cp config.example.json config.json
```

*Отредактируйте `config.json`, добавив необходимые пути, настройки парсера и API-ключи для ИИ-сервисов.*

### Запуск системы

Проект включает в себя `.bat` файлы для удобного запуска под Windows:

* **`start_ASSE.bat`** — Запуск фонового планировщика (`autostart.py`), который управляет ETL-процессами.
* **`run_nicegui.bat`** (или `run_dashboard.bat`) — Запуск графического интерфейса пользователя (Дашборда).

## 🛠 Технологический стек

* **Backend:** Python, SQLite, Pandas
* **Frontend:** NiceGUI, Streamlit (legacy)
* **AI & OCR:** OpenAI API / Google Gemini API, difflib (алгоритм Ратклиффа-Обершельпа)
